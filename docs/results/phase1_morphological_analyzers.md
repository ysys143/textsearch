## 핵심 질문과 답

**질문:** 한국어 BM25 검색에서 어떤 형태소 분석기가 최고 품질을 제공하는가?

**답:** **kiwi-cong** (Kiwi with conjugation) — 두 데이터셋 모두에서 NDCG@10 최고점.

## 최종 선정 분석기

### 우선순위 1: **kiwi-cong**

**선정 이유:**
- MIRACL에서 최고 NDCG@10 (0.3471) — +42.6% vs 기준선
- EZIS에서 최고 NDCG@10 (0.9455) — +13.2% vs 기준선
- MRR도 양쪽 데이터셋에서 최고
- 처리량 합리적 (241 docs/s) — 실시간 인덱싱 가능
- **Phase 2, 3, 4의 기본 분석기로 권장**

### 우선순위 2: **mecab**

**선정 이유:**
- MIRACL에서 세 번째 (NDCG@10 0.3147) — +43% vs 기준선
- EZIS에서 두 번째 (NDCG@10 0.9124) — +9.2% vs 기준선
- 최고 처리량 (587 docs/s) — 대규모 인덱싱 최적
- C extension 기반, 안정적, 잘 알려진 도구
- **속도가 중요한 환경에서 kiwi-cong 대안**

### 우선순위 3: **OKT** (선택적)

**선정 이유:**
- MIRACL에서 두 번째 (NDCG@10 0.3333)
- 좋은 품질, 그러나 느림 (62 docs/s, JVM 오버헤드)
- **선택적으로만 포함, 성능 비교용**

## 제외 분석기

### Kkma
- MIRACL: NDCG@10 0.3268 (+48.5% vs 기준선)
- EZIS: NDCG@10 0.9056 (+8.4% vs 기준선)
- **문제:** 극히 느림 (5-8 docs/s)
- **결론:** 실전 사용 부적합, **제외 권장**

### kiwi-knlm
- **제외 이유:** `sj.knlm` 모델 파일 미포함 (번들되지 않음)
- 재평가 필요시 별도 작업

### khaiii (Kakao)
- **제외 이유:** ARM64 빌드 실패 (cmake HunterGate 오류, PyPI 휠 없음)
- Apple Silicon 환경에서 사용 불가

## Phase 2로의 시사점

1. **형태소 분석기 선정:** kiwi-cong + mecab 이중 설정
   - kiwi-cong: 기본값 (최고 품질)
   - mecab: 속도 중심 배포 대안

2. **어휘 수 최적화:**
   - kiwi-cong: 11,829개 (MIRACL) vs 1,678개 (EZIS) — 데이터셋별 편차 큼
   - BM25 파라미터 튜닝에 영향 가능

3. **Query 레이턴시:**
   - kiwi-cong: 6.94ms (MIRACL) — 대규모 쿼리에 실용적
   - OKT의 느림은 단순 속도 문제 아님 (좋은 품질 대가)

4. **재현율 분석:**
   - MIRACL: 모든 분석기 < 40% (일부 쿼리만 관련 문서 포함)
   - EZIS: 모든 분석기 > 94% (매우 높은 재현율)

5. **다음 검증:**
   - Phase 2: BM25 파라미터 (k1, b) 튜닝
   - Phase 3: 인덱싱 전략 (컨텍스트 윈도우, 청킹)
   - Phase 4: 리랭킹 (BM25 + Dense retrieval)

---

## 결론

한국어 BM25 검색에서 **kiwi-cong**이 최고 품질을 제공하며, **mecab**이 속도와 품질의 합리적 균형을 제시한다. Phase 2부터는 이 두 분석기를 중심으로 진행한다.
